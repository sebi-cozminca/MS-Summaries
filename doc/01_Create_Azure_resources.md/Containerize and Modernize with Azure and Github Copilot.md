## Exercise Summary

**Exercise 1: Setup the Environment**
- **Goal:** Prepare the lab environment and authenticate with required services for migration.
- **Actions:**
  - Log in to the lab VM and GitHub SSO portal with provided credentials.
  - Open the project in Visual Studio Code and configure GitHub Copilot and Git settings.
- **Validation:**
  - Successful sign-in to GitHub and VS Code; project folder is open and ready for assessment.

**Exercise 2: Assess Your Java Application**
- **Goal:** Evaluate the asset-manager Java application for cloud readiness and migration blockers.
- **Actions:**
  - Use the GitHub Copilot App Modernization extension to run an automated assessment.
  - Review the generated assessment report and proposed solutions for migration issues.
- **Validation:**
  - Assessment report is generated and issues are identified with recommended solutions.

**Exercise 3: Migrate to Azure Database for PostgreSQL Flexible Server**
- **Goal:** Migrate the application's database layer from AWS PostgreSQL to Azure Database for PostgreSQL using managed identity.
- **Actions:**
  - Select and run the migration task for PostgreSQL in the Copilot App Modernization extension.
  - Review and accept code changes and migration summary generated by Copilot.
- **Validation:**
  - Code changes for database migration are applied and summarized in the migration report.

**Exercise 4: Install and Authorize GitHub Copilot CLI**
- **Goal:** Enable command-line driven modernization tasks using GitHub Copilot CLI.
- **Actions:**
  - Install GitHub Copilot CLI and PowerShell 7 using winget.
  - Authorize the CLI with GitHub credentials and verify device activation.
- **Validation:**
  - Copilot CLI is installed, authorized, and ready for use in the project directory.

**Exercise 5: Migrate from AWS S3 to Azure Blob Storage**
- **Goal:** Modernize the application's storage layer by migrating from AWS S3 to Azure Blob Storage.
- **Actions:**
  - Use Copilot CLI to prompt and execute migration from S3 to Azure Blob Storage.
  - Review and accept code changes as prompted by Copilot.
- **Validation:**
  - Storage migration code changes are applied and confirmed by Copilot CLI output.

**Exercise 6: Migrate from AMQP RabbitMQ to Azure Service Bus**
- **Goal:** Replace RabbitMQ messaging with Azure Service Bus using managed identity.
- **Actions:**
  - Run the messaging service migration task in the Copilot App Modernization extension.
  - Review and accept code changes and migration summary.
- **Validation:**
  - Messaging migration code changes are applied and summarized in the migration report.

**Completion**
- **Goal:** Finalize and validate the modernization process.
- **Actions:**
  - Review migration summary and plan files for details and next steps.
  - Confirm all major migrations (database, storage, messaging) are complete and code is updated.
- **Validation:**
  - All migration steps are completed; code is modernized and ready for further testing or deployment.
